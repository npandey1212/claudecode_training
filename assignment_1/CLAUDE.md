# Pydantic Codebase Exploration

## Mission
Understand architecture, trace validation pipeline, find bugs, generate diagrams.

## Rules
- Never hallucinate. If unsure, say "I'm not certain".
- Always reference exact file paths and line numbers.
- Prefer reading actual source over assumptions.
- When tracing, follow actual function calls — don't summarize abstractly.

## Goals
1. Summarize the project purpose
2. Map top-level directory structure
3. Identify main entry points
4. Generate architecture, sequence, and ER diagrams in Mermaid
5. Trace a full request with file paths and line numbers
