Validation Flow

What happens when you call MyModel(field=value) — click each step to expand

1
User calls MyModel(field=value) pydantic/main.py ▶
Python's normal constructor dispatch — no magic yet. The class was built by ModelMetaclass which already compiled the schema.
# pydantic/main.py : 253 def __init__(self, /, **data: Any) -> None: self.__pydantic_validator__.validate_python(data, self_instance=self)
2
SchemaValidator.validate_python() called pydantic_core (Rust) ▶
__pydantic_validator__ is a SchemaValidator (or PluggableSchemaValidator) created once at class definition time. It holds the compiled CoreSchema.
# pydantic/main.py : 204 __pydantic_validator__: ClassVar[SchemaValidator | PluggableSchemaValidator] # Created in pydantic/_internal/_model_construction.py : 690 cls.__pydantic_validator__ = create_schema_validator( schema, cls, module, qualname, 'BaseModel', core_config, plugin_settings )
3
CoreSchema executed field-by-field pydantic_core (Rust) ▶
The Rust engine walks the model_schema tree, validating each field against its schema. Validators (before/wrap/after) are woven in as function-call nodes.
# Schema shape built by _generate_schema.py { "type": "model", "cls": MyModel, "schema": { "type": "model-fields", "fields": { "name": { "type": "model-field", "schema": { "type": "str" } }, "age": { "type": "model-field", "schema": { "type": "int" } }, } } }
4
Schema was built by GenerateSchema pydantic/_internal/_generate_schema.py ▶
At class definition time, ModelMetaclass.__new__ triggers schema generation. This is the heavy lifting — it runs once, not on every validation call.
# _generate_schema.py — main dispatch def _generate_schema_inner(self, obj): # L1007 if lenient_issubclass(obj, BaseModel): return self._model_schema(obj) return self.match_type(obj) # L1035 — handles all other types # match_type checks __get_pydantic_core_schema__ for custom types def match_type(self, obj): # L1035 if obj is str: return core_schema.str_schema() if obj is int: return core_schema.int_schema() # ... more built-ins ... return self._generate_schema_from_get_schema_method(obj) # L891
Rust validation completes — two outcomes:
✅ Success path
Rust returns validated Python object.

__init__ returns normally.
Fields are set on self.__dict__ (or __pydantic_fields_set__).
❌ Failure path
Rust raises pydantic_core.ValidationError.

Re-exported as pydantic.ValidationError.
Contains structured list of InitErrorDetails per failing field.

At class definition, ModelMetaclass scans all methods for decorator markers.

Stored on the class as __pydantic_decorators__ (a DecoratorInfos instance).

 • validators — V1-style
 • field_validators — V2 @field_validator
 • model_validators — V2 @model_validator
 • field_serializers — @field_serializer
